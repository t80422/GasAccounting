# 銀行月結餘額重整工具使用說明

## 📋 功能說明

此工具用於重新計算所有銀行的月結餘額，適用於以下情境：
- 資料異常需要修復
- 系統升級後的資料校驗
- 定期維護作業

## 🎯 執行方式

### 方式一：透過程式碼調用（建議在主選單或工具選單中加入）

在 `frmMain` 中已新增方法，可以透過以下方式調用：

```vb
' 在 frmMain 的任何事件中調用
Me.OpenBankBalanceRecalculator()
```

### 方式二：直接開啟表單

```vb
Dim recalculatorForm As New frmBankBalanceRecalculator()
recalculatorForm.ShowDialog()
```

## ⚙️ 執行流程

1. **開啟工具**
   - 工具會顯示警告訊息，提醒使用者此操作的影響

2. **確認執行**
   - 點擊「執行重整」按鈕
   - 系統會再次確認是否要繼續

3. **執行重整**
   - 系統會：
     - 刪除所有現有的銀行月結餘額記錄
     - 從所有 payment 和 collection 交易中重新計算
     - 逐月重建每個銀行的月結餘額
     - 自動處理連鎖效應（後續月份的期初期末餘額）

4. **查看結果**
   - 顯示處理成功的銀行數量
   - 如有錯誤，會顯示詳細的錯誤訊息

## 🔧 技術細節

### 重整邏輯

```vb
' Service 層提供的方法
Function RecalculateAllBanksAsync(
    bmbRep As IBankMonthlyBalancesRep,
    bankRep As IBankRep,
    paymentRep As IPaymentRep,
    collectionRep As ICollectionRep
) As Task(Of String)
```

### 處理步驟

1. **取得所有銀行**
   ```vb
   Dim allBanks = Await bankRep.GetAllAsync()
   ```

2. **對每個銀行執行重整**
   - 取得該銀行所有交易月份
   - 刪除舊的月結記錄
   - 逐月重新計算

3. **月結計算公式**
   ```
   期末餘額 = 期初餘額 + 收入總額 - 支出總額
   下月期初 = 本月期末
   ```

## ⚠️ 注意事項

### 執行前

1. **備份資料庫**
   - 此操作會刪除所有現有月結記錄
   - 建議先備份資料庫

2. **確認時機**
   - 建議在非營業時間執行
   - 確保沒有其他使用者正在操作

3. **檢查權限**
   - 確認執行者有足夠的權限

### 執行中

1. **不要中斷**
   - 執行過程中請勿關閉程式
   - 等待進度條完成

2. **注意訊息**
   - 觀察結果文字框中的訊息
   - 如有錯誤立即記錄

### 執行後

1. **驗證結果**
   - 檢查重整後的月結餘額是否正確
   - 比對幾個關鍵月份的數據

2. **測試功能**
   - 測試付款、收款功能是否正常
   - 確認增量更新功能運作正常

## 🐛 常見問題

### Q1: 重整失敗怎麼辦？

**A:** 
- 系統會自動 Rollback，不會影響原有資料
- 查看錯誤訊息，找出問題所在
- 修復問題後再次執行

### Q2: 重整需要多久時間？

**A:** 
- 取決於資料量
- 通常幾秒到幾分鐘不等
- 系統會顯示進度條

### Q3: 可以只重整特定銀行嗎？

**A:** 
- 目前工具是重整所有銀行
- 如需單一銀行重整，可以使用：
  ```vb
  Await bmbService.RecalculateBankBalancesAsync(
      uow.BankMonthlyBalancesRepository,
      uow.BankRepository,
      uow.PaymentRepository,
      uow.CollectionRepository,
      bankId  ' 指定銀行ID
  )
  ```

### Q4: 重整會影響其他資料嗎？

**A:** 
- 不會，只影響 `bank_monthly_balances` 資料表
- 不會修改 payment 或 collection 資料
- 使用 Transaction 保證資料一致性
